Biblioteca de Componentes

Documentação técnica dos elementos de interface do Fóton Framework.

Botões

Utilize a classe base .btn seguida de um modificador de estilo. A tag pode ser <button> ou <a>. Os botões possuem transições suaves, estados de hover automáticos e reset de estilos nativos.

<!-- Botão com gradiente da marca (Destaque Principal) --> <button class="btn btn-gradient">Ação Principal</button> <!-- Botão secundário/neutro (Fundo cinza claro) --> <button class="btn btn-primary">Cancelar</button> <!-- Botão sólido escuro (Contraste alto) --> <button class="btn btn-dark">Dark</button>

Botões Semânticos

Use cores semânticas para indicar o feedback ou propósito da ação (sucesso, aviso, perigo, informação). Adicione a classe correspondente ao estado desejado.

<!-- Classes Semânticas --> <button class="btn btn-success">Success</button> <button class="btn btn-warning">Warning</button> <button class="btn btn-danger">Danger</button> <button class="btn btn-info">Info</button>

Botões Outline (Borda)

Para interfaces mais leves e secundárias, use as variações .btn-outline-*. Inclui variantes translúcidas (White/Dark) para uso sobre fundos coloridos ou imagens.

<!-- Outlines Coloridos --> <button class="btn btn-outline-success">Success</button> <button class="btn btn-outline-warning">Warning</button> <button class="btn btn-outline-danger">Danger</button> <button class="btn btn-outline-info">Info</button> <!-- Translúcidos (Fundo Transparente) --> <button class="btn btn-outline-dark">Dark Outline</button> <button class="btn btn-outline-white">White Outline</button>

Links e Tamanhos

Adicione .btn-sm para botões compactos (padding reduzido e fonte menor) ou .btn-block para ocupar 100% da largura do container pai (display flex). Para links que parecem texto mas têm comportamento de botão, use .btn-link.

Botão Bloco (.btn-block)

Toggle Switch

Um checkbox estilizado como interruptor físico. A estrutura requer um <label> envolvendo o input checkbox oculto e uma div .toggle__btn que renderiza o visual.

<label class="toggle__section"> <input type="checkbox"> <div class="toggle__btn"><span></span></div> <span>Label do Switch</span> </label>

Alertas

Componentes de feedback para mensagens de usuário. Utilize a classe .alert combinada com uma cor contextual (ex: .alert-warning). O botão de fechar requer a classe .alert-close e a funcionalidade de "dismiss" é gerenciada automaticamente pelo script foton_scripts.js.

Operação Realizada Seus dados foram salvos com sucesso no banco de dados.
Atenção: Sua assinatura expira em 3 dias.
Erro Crítico: Falha ao conectar ao servidor.
Info: Nova versão disponível para download.
<div class="alert alert-primary"> <!-- Ícone opcional --> <i class="fa-solid fa-check alert-icon"></i> <!-- Conteúdo --> <div> <span class="alert-heading">Título</span> Mensagem de corpo. </div> <!-- Botão de fechar (JS automático) --> <button class="alert-close"><i class="fa-solid fa-xmark"></i></button> </div>

Badges

Indicadores visuais para status, contadores ou categorias. A classe base .badge aplica padding e alinhamento. Combine com cores contextuais (ex: .badge-success) para alterar o fundo e a cor do texto. Adicione .badge-pill para bordas totalmente arredondadas (estilo pílula).

Primary Secondary Success Warning Danger Info Neutral
Pill Style Ativo 12

Avatares

Componente para imagens de perfil. A classe base .avatar cria um círculo com overflow hidden. Use as classes de tamanho para ajustar as dimensões: .avatar-sm (32px), .avatar-md (48px), .avatar-lg (64px) ou .avatar-xl (96px). Para agrupar avatares, envolva-os em um container .avatar-group.

Tabelas

Tabelas leves e limpas. Adicione a classe .table na tag <table>. Modificadores como .table-hover (destaque ao passar o mouse) e .table-striped (listrada) estão disponíveis. Envolva em .table-responsive para habilitar rolagem horizontal em dispositivos móveis sem quebrar o layout.

Navegação & UI Elements

Breadcrumbs

Auxilia na navegação hierárquica do site. Utilize a classe .breadcrumb em uma lista ordenada (ol) dentro de um elemento nav com aria-label="breadcrumb" para acessibilidade. Itens ativos devem receber a classe .active.

  1. Home
  2. Biblioteca
  3. Dados
<nav aria-label="breadcrumb"> <ol class="breadcrumb"> <li class="breadcrumb-item"><a href="#">Home</a></li> <li class="breadcrumb-item active">Atual</li> </ol> </nav>

Dropdowns

Menus flutuantes acionáveis. O framework suporta dropdowns via CSS (hover) mas recomenda-se o uso do atributo data-toggle="popover" no botão gatilho para melhor suporte em dispositivos touch e controle via JavaScript (clicar fora para fechar). A estrutura exige um container .dropdown envolvendo o botão e a lista .dropdown-menu.

  • Editar
  • Duplicar
  • Excluir
<div class="dropdown"> <!-- Gatilho com ícone --> <button class="btn btn-primary" data-toggle="popover"> Menu <i class="fa-solid fa-chevron-down"></i> </button> <!-- Lista de Itens --> <ul class="dropdown-menu"> <li><a class="dropdown-item" href="#">Item 1</a></li> <li><a class="dropdown-item" href="#">Item 2</a></li> <!-- Divisor --> <li style="border-top: 1px solid #eee;"></li> <li><a class="dropdown-item" href="#">Sair</a></li> </ul> </div>

Cards

Container flexível e extensível com borda, sombra suave e padding padrão. Use .card como wrapper principal e .card-body para o conteúdo interno, garantindo espaçamento consistente.

1. Card Padrão & Destaque

Padrão

Card básico com sombra suave.

Ação

Destaque

Use .card-highlight para bordas coloridas e tag "Popular".

Ação
<!-- Card Simples --> <div class="card"> <div class="card-body"> <h3>Título</h3> <p>Conteúdo...</p> </div> </div> <!-- Card Destacado (Borda colorida + Tag Popular) --> <div class="card card-highlight"> <div class="card-body">...</div> </div>

2. Variações Específicas

Classes auxiliares para casos de uso comuns que alteram o layout do card:

  • .card-blog: Remove o padding do topo para permitir imagens "full-width" (largura total) no cabeçalho.
  • .card-profile: Centraliza todo o conteúdo e adiciona espaçamentos ideais para avatares e nomes.
  • .card-feature-row: Altera o layout para horizontal (flex-row), ideal para listar funcionalidades com ícones à esquerda.

Acordeões

Componente expansível para FAQs ou conteúdo colapsável. Baseado na tag nativa <details> e <summary> para máxima semântica e acessibilidade. O arquivo JavaScript do framework adiciona automaticamente uma animação suave de altura (slide down/up) ao abrir e fechar, garantindo uma experiência fluida (60fps).

Como funciona a animação?
O JavaScript calcula a altura do conteúdo e transiciona a propriedade height, oferecendo uma experiência de 60fps.
É necessário JavaScript?
O componente funciona sem JS (abre/fecha instantaneamente), mas o script incluído adiciona a animação de deslize suave.
<details class="accordion"> <summary>Pergunta ou Título</summary> <div class="accordion-body"> Conteúdo oculto que será expandido... </div> </details>

Popovers & Tooltips

Elementos flutuantes para informações contextuais. Tooltips são ativados via hover e implementados com CSS puro usando o atributo data-tooltip. Popovers são ativados via clique e requerem a estrutura HTML completa e o atributo data-toggle="popover" no botão para que o JavaScript gerencie a abertura e o fechamento ao clicar fora.

Estrutura de Página

Componentes estruturais para montar o layout macro do seu site.

Navbar (Menu Superior)

A barra de navegação é fixa no topo por padrão (position: fixed no CSS do framework). Utilize um container flex interno para alinhar o logo e os links. O exemplo abaixo simula a visualização.

Hero Section

Área de destaque inicial. Possui padding generoso (padding-top ajustado para compensar a navbar fixa) e centralização vertical/horizontal do conteúdo dentro de .hero-content.

Título de Impacto

Subtítulo explicativo curto para engajar o usuário.

<section class="hero"> <div class="hero-content"> <span>Título Principal</span> <p>Descrição...</p> <button class="btn btn-gradient">Ação</button> </div> </section>

Footer

Rodapé simples. Geralmente usado dentro de um container com cor de fundo diferente ou como último elemento do corpo da página.
